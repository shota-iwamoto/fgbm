FORT-GBM v2 基本設計書

Financial Ordered Robust Student-t Gradient Boosting Machine

Student-tロバスト学習、時系列ordered encoding、era-level DRO、 era-robust split、因子中立化、turnover正則化を統合する 金融特化型勾配ブースティングライブラリの基本設計を定義する。

本書はライブラリの責務、構成、主要インターフェース、学習フロー、 データ構造、制約および受入条件を定める基本設計書である。 個々の数値アルゴリズム、メモリ配置、SIMD、GPU kernelなどは詳細設計で確定する。

1. 目的と適用範囲

1.1 開発目的

FORT-GBM v2は、金融時系列・横断面データに特有の次の問題を、 後処理ではなく学習アルゴリズム内部で制御する。

  1. 極端値、誤ラベル、fat-tail residual
  2. 特定eraまたは特定regimeへの性能集中
  3. カテゴリtarget encodingによる時系列リーク
  4. 既知risk factorとalphaの混同
  5. 低era coverageの葉による偶然適合
  6. 予測変動に起因する過剰turnover
  7. 平均性能だけを基準とする不安定なモデル選択

1.2 対象タスク

  • 回帰
  • 二値分類
  • 多クラス分類
  • pairwise ranking
  • NDCG指向ranking
  • quantile回帰

1.3 対象外

  • 注文執行アルゴリズム
  • 完全なportfolio optimization
  • バックテスト約定シミュレーション
  • リアルタイム特徴量配信基盤
  • 未知の分布変化に対する無条件の性能保証

2. 設計原則

原則 設計方針
ロバスト性 Student-t objectiveを標準とし、MM majorizationで安定した正曲率を構成する。
era中心 サンプル平均ではなく、era内正規化損失とera-level DROを基本単位とする。
リーク防止 カテゴリ統計はtarget maturity、purge、embargoを満たす過去データだけから生成する。
目的関数整合性 分割、葉値、木の受理を可能な限り同一の統合objectiveに整合させる。
保守的木探索 全体利得だけでなく、era別利得の下側DRO値とera coverageを評価する。
運用可能性 factor exposureと予測turnoverを学習正則化へ組み込む。
安全な近似 近似Hessianで木を生成した後、完全objectiveのline searchで受理を決定する。
再現性 データ順序、乱数seed、fold、binning、pair samplingをモデル成果物へ保存する。

8. 統合目的関数

FORT-GBM v2の基本objectiveを次とする。

J(F, σ, ν) = maxq ∈ Qρ,κ Σe qeLe(F; σ, ν)
+ λdownRdown(F) + λfactorRfactor(F) + λturnRturn(F)
+ λscaleRscale(σ) + ΣmΩ(fm).
項 責務
\(L_e\) era内正規化されたtask loss
\(Q_{\rho,\kappa}\) KL半径とera weight capを持つambiguity set
\(R_{\mathrm{down}}\) 固定benchmarkより悪いeraを抑制
\(R_{\mathrm{factor}}\) 予測の既知factor射影を抑制
\(R_{\mathrm{turn}}\) 同一instrumentの時系列予測変化を抑制
\(R_{\mathrm{scale}}\) era scaleの過剰変動とcollapseを防止
\(\Omega\) 葉数、深さ、葉値、coverageを正則化

8.1 objectiveの評価モード

  • surrogate evaluation： 木探索用の勾配・正曲率近似を高速計算する。
  • exact evaluation： 木追加候補に対して完全objectiveを再計算する。
  • validation evaluation： out-of-timeデータ上で金融評価指標を計算する。

10. Era-DRO Solver

10.1 責務

  • era損失ベクトルの受領
  • 基準era分布の構成
  • KL半径とweight capの制約処理
  • 最悪分布 \(q^\star\) の計算
  • primal、dual、KKT residualの返却

10.2 ambiguity set

Qρ,κ = { q ≥ 0 : Σeqe = 1, DKL(q || π) ≤ ρ, qe ≤ κπe }.

10.3 出力

EraDROResult
├── robust_risk
├── adversarial_era_weight
├── kl_divergence
├── active_weight_caps
├── primal_residual
├── dual_residual
└── convergence_status

10.4 エラー条件

  • 基準重み \(\pi_e\) が0以下
  • weight capの総容量が1未満
  • KL半径が負
  • era損失にNaNまたは無限値が存在
  • 指定反復数内にKKT residualが許容値へ収束しない

11. Ordered Encoding

11.1 利用可能な過去集合

Pi = { j : target_maturityj + embargo < observation_timei }.

sample \(i\) のカテゴリ統計は、必ず \(P_i\) の観測だけで構築する。

11.2 エンコード種別

  • 過去count
  • 時間減衰count
  • 平滑化target mean
  • Student-t robust location
  • 分類class frequency
  • ranking relevance statistic
  • カテゴリ組合せ統計

11.3 未知カテゴリ

推論時の未知カテゴリは次の順序で処理する。

  1. カテゴリ階層の親統計
  2. 低次カテゴリ組合せ統計
  3. era非依存global prior

12. Financial Regularizer

12.1 Factor penalty

eraごとに中心化したfactor matrixへ予測を射影し、 射影成分の加重二乗ノルムを罰する。

Rfactor(F) = 1/2 Σe πesF,e-2 (CeFe)T WePe (CeFe).

factor operatorは学習foldだけから構成し、当該木の構築中は固定する。

12.2 Turnover penalty

固定線形portfolio operatorを \(B_e\) とし、 ポジションscoreを \(p_e=B_eF_e\) とする。

Rturn(F) = Σe,a ca,e [ √(da,e2 + δ2) - δ ].

\(d_{a,e}\) は同一instrumentの連続era間におけるposition score差である。

12.3 責務分離

処理 FORT-GBM内 外部portfolio optimizer
因子露出penalty 対象 必要に応じて追加制約
予測turnover proxy 対象 実ポジションturnoverを再最適化
gross・net exposure 線形近似のみ 厳密制約
market impact 係数による近似 非線形費用モデル
position limit 対象外 厳密制約

13. Tree Builder

13.1 木成長方式

leaf-wise成長を基本とし、最大深さ、最大葉数、 最小曲率、最小era coverageによって制限する。

13.2 Histogram統計

各feature bin、node、eraについて次を保持する。

EraHistogramBin
├── gradient_sum
├── curvature_sum
├── sample_count
├── sample_weight_sum
├── recent_era_flag
└── missing_value_count

era別統計を全binへ無制限に保持するとメモリ量が増大するため、 詳細設計では次のモードを提供する。

  • exact-era：全era統計を保持
  • era-block：隣接eraをblock化
  • top-era：主要eraと残余集約を保持

標準仕様および検証基準はexact-eraとする。

13.3 Missing value

欠損値は独立binとして集計し、左・右の両方向を候補評価する。 欠損方向もera-robust split scoreに基づいて決定する。

13.4 カテゴリ分割

カテゴリはordered statisticまたはカテゴリbinへ変換し、 target maturity条件を満たす状態だけを使う。 生target平均による全期間カテゴリ並べ替えは禁止する。

14. Robust Split Evaluator

14.1 葉統計

GA,e = Σi∈A,ei=eGi,    HA,e = Σi∈A,ei=eHi.

14.2 Era effective coverage

uA,e = HA,e / (HA + ε),    Neff,A = 1 / (ΣeuA,e2 + ε).

sample countではなく曲率質量を使用し、 objective上の情報量が一部eraへ集中する葉を検出する。

14.3 Coverage prior

λA,total = λ2 + λA,coverage.

葉値は次の凸部分問題の解とする。

vA = -SoftThreshold(GA, λ1) / (HA + λA,total).

14.4 Era別改善量

親葉と左右子葉の共通葉値を用いて、各eraのsurrogate改善量 \(\Delta_e\) を計算する。

14.5 Robust lower gain

Grobust = minq∈Qsplit ΣeqeΔe.

14.6 Split score

Score = Grobust - ΔΩ - κnegΣeπe(-Δe)+ - κdispMAD(Δ).

14.7 Split採用条件

  • split scoreが正
  • 左右の総曲率が下限以上
  • 左右のeffective era coverageが下限以上
  • 最近eraの支持率が下限以上
  • 最大深さと最大葉数の範囲内
  • monotonic constraintに違反しない
  • interaction constraintに違反しない

15. 学習制御

15.1 1回のboosting反復

1. 現在予測を固定
2. Student-t scale・自由度を必要に応じて更新
3. era損失を計算
4. capped KL-DROを解く
5. task gradientとcurvatureを計算
6. factor・turnover・downsideの勾配を合成
7. PSDまたは対角majorizerを構成
8. histogramを生成
9. era-robust splitで木を構築
10. coverage priorから葉値を計算
11. 完全objective上でline search
12. 木を受理または棄却
13. validation指標を更新
14. early stoppingを判定

15.2 Line search

α* = arg min0≤α≤αmax J(F + αf).

標準実装はbacktracking Armijo searchとする。

  1. 初期stepを設定する
  2. 候補予測を生成する
  3. DRO内側問題を再最適化する
  4. 完全objectiveを評価する
  5. Armijo条件を満たさなければstepを縮小する
  6. 最小step未満なら木を棄却する

15.3 Rollback

木の受理前に次をcheckpointする。

  • 予測ベクトル
  • Student-t scale・自由度
  • DRO状態
  • ordered stateの参照version
  • training history

17. ValidationとEarly Stopping

17.1 検証方式

purged walk-forwardを標準方式とする。

Traink = { i : target_maturityi + embargo < validation_startk }.

17.2 主要指標

  • 平均era loss
  • worst-era loss
  • era-CVaR
  • era correlation
  • feature-neutral correlation
  • era scoreのHAC補正Sharpe
  • 最大drawdown
  • factor exposure
  • turnover
  • 費用控除後utility

17.3 Early stopping score

Uval = mean(Se) - κstdstd(Se) - κtailCVaR(-Se) - κturnTurnover - κfactorExposure.

複数foldの平均値ではなく、中央値または下側信頼限界を標準選択基準とする。

17.4 禁止事項

  • ランダムK-foldによる金融時系列評価
  • validation targetを用いたカテゴリ統計更新
  • validation期間を用いたfactor operator推定
  • validation結果に基づく無制限の反復的特徴量選択

18. 推論設計

18.1 Batch prediction

  1. 入力schemaを検証する
  2. 保存済みbin境界を適用する
  3. 利用可能時点のordered encoding stateを適用する
  4. 全木の葉値を加算する
  5. task linkを適用する
  6. 任意でfactor neutralizationを適用する
  7. 任意でera内rank変換を適用する

18.2 Online state update

targetが成熟した時点でのみordered stateを更新する。 

observe features
    |
predict
    |
wait until target maturity
    |
receive target
    |
validate maturity and ordering
    |
update ordered encoding state

18.3 Point-in-time再現

任意の過去時点における予測を再現できるよう、 ordered stateにversionと有効時刻を保持する。

21. エラー処理

分類 例 処理
入力不正 era欠落、負のweight、時刻不整合 学習開始前に停止
リーク危険 target maturityが観測時刻以前に定義不能 原則停止
数値不安定 NaN、無限大、scale collapse rollback後に停止または安全設定で再試行
DRO未収束 KKT residual超過 反復延長後も未収束なら木を棄却
line search失敗 最小stepでもobjective非減少 木を棄却し、棄却回数を記録
validation劣化 utility LCBが継続悪化 early stoppingし最良iterationへ戻す

  • 疎factor matrix対応
  • instrument対応行列の疎表現
  • DRO solverのwarm start
  • line search候補の予測差分更新

  • CPU並列を初期標準とし、GPUは後続段階で追加

23. セキュリティと再現性

  • モデル読込時にschema versionとchecksumを検証する
  • 任意コード実行を伴うserialization形式を標準にしない
  • 学習データの行identifierをモデルへ保存しない
  • 乱数生成器とseedを明示的に管理する
  • 決定論モードでは並列reduce順序を固定する
  • 全設定と自動補正値をtraining manifestへ保存する

  • 単一era汚染時のDRO weight cap
  • 低coverage偽signalの分割拒否

24.5 回帰テスト

  • 固定seedでモデルchecksumが一致
  • 既知データでvalidation metricが許容範囲内
  • 性能最適化後も数学テスト結果が不変

27. v2基本仕様の確定事項

  1. 回帰の標準objectiveはStudent-tとする。
  2. Student-t木構築にはMM majorizerを使用する。
  3. era損失はera内で正規化する。
  4. DROはKL半径と敵対era weight capを持つ。
  5. カテゴリ統計はtarget maturityを満たすstrict-pastデータだけで作る。
  6. splitはera別改善量のrobust lower gainで評価する。
  7. 葉縮小はcoverage-dependent priorとして目的関数へ組み込む。
  8. factor penaltyはPSD二次形式とする。
  9. turnover penaltyは固定線形operator上の凸smooth-L1とする。
  10. 近似木は完全objectiveのline searchで受理する。
  11. validationはpurged walk-forwardを標準とする。
  12. 学習状態、時系列状態、数値設定をモデル成果物へ保存する。
FORT-GBM v2の中心設計は、 「Student-tでサンプル外れ値を抑え、capped era-DROでregime集中を抑え、 ordered learningで未来リークを防ぎ、robust splitとcoverage priorで木構造の偶然適合を抑え、 factor・turnover正則化で金融運用上の制約を学習へ統合し、 exact line searchで近似更新の破綻を拒否する」 という一貫した構造である。
