FORT-GBM v2 詳細設計書

Financial Ordered Robust Student-t Gradient Boosting Machine

Student-tロバスト尤度、target-maturity-aware ordered encoding、 capped era-level KL-DRO、era-robust split、coverage-dependent leaf prior、 factor-neutral penalty、turnover regularizationおよび完全目的関数line searchを 統合する金融特化型勾配ブースティングライブラリの詳細設計を定義する。

本書における「完全目的関数」は、設定された学習データ、固定済み演算子、 当該木について確定済みの構造正則化係数、補助パラメータおよびDRO内側最適化を含む 実装上の目的関数を意味する。未知の将来分布に対する無条件の汎化保証を意味しない。

1. 文書の目的

本書は基本設計を実装可能な粒度へ分解し、以下を確定する。

  • モジュール、クラス、データ構造および依存関係
  • 目的関数、勾配、曲率および補助パラメータ更新
  • capped KL-DROの数値解法
  • ordered categorical statisticsのpoint-in-time処理
  • ヒストグラム、分割探索、葉値計算および木構築
  • factor penaltyとturnover penaltyの演算方式
  • 完全目的関数line searchとrollback
  • 回帰、分類、rankingおよびquantileのタスク固有処理
  • 永続化、並列化、数値安定化、テストおよび受入条件

6.4 Era内正規化重み

\[ W_e=\sum_{i:e_i=e}\omega_i, \qquad a_i=\frac{\omega_i}{W_{e_i}}. \]

以下を不変条件とする。

\[ \omega_i\geq0, \qquad W_e>0, \qquad \sum_{i:e_i=e}a_i=1. \]

6.5 時系列リーク検証

過去集合は、

\[ \mathcal P_i = \left\{ j: t_j^{\mathrm{maturity}} + \delta_{\mathrm{embargo}} < t_i^{\mathrm{observation}} \right\} \]

とする。境界の等号を許可するかは市場データの公開時刻仕様で設定する。

era番号の大小だけで過去集合を判定してはならない。 target horizonが重なる場合、過去eraのtargetでも予測時点では未成熟である可能性がある。

9. Student-t回帰実装

9.1 損失

\(r_i=y_i-F_i\)、\(A_i=\nu\sigma_i^2+r_i^2\) とする。

\[ \ell_i = \log\sigma_i + \frac12\log(\nu\pi) + \log\Gamma\left(\frac{\nu}{2}\right) - \log\Gamma\left(\frac{\nu+1}{2}\right) + \frac{\nu+1}{2} \log\left(1+\frac{r_i^2}{\nu\sigma_i^2}\right). \]

9.2 勾配とMM曲率

\[ g_i = -\frac{(\nu+1)r_i}{A_i}, \qquad h_i^{\mathrm{MM}} = \frac{\nu+1}{A_i}. \]

実装値は、

\[ h_i = \min \left( h_{\max}, \max(h_{\min},h_i^{\mathrm{MM}}) \right). \]

9.3 Scale更新

scale更新はeraごとの負対数尤度と階層正則化を最小化する。 \(s_e=\log\sigma_e\) と置き、下限を、

\[ \sigma_e=\sigma_{\min}+\operatorname{softplus}(u_e) \]

で保証する。

scaleに関する微分は、

\[ \frac{\partial\ell_i}{\partial s_e} = 1 - \frac{(\nu+1)r_i^2} {\nu\sigma_e^2+r_i^2} \]

である。実装ではdamped NewtonまたはL-BFGSを用い、 更新前後の完全なscale部分目的関数を比較する。

9.4 自由度更新

\(\nu\) は制約付き一次元最適化で更新する。 微分にはdigamma関数を使用する。

\[ \begin{aligned} \frac{\partial\ell_i}{\partial\nu} ={}& \frac{1}{2\nu} + \frac12\psi\left(\frac{\nu}{2}\right) - \frac12\psi\left(\frac{\nu+1}{2}\right) \\ &+ \frac12 \log\left(1+\frac{r_i^2}{\nu\sigma_i^2}\right) - \frac{(\nu+1)r_i^2} {2\nu(\nu\sigma_i^2+r_i^2)}. \end{aligned} \]

bounded Brent法を標準とし、 \([\nu_{\min},\nu_{\max}]\) の範囲で補助目的関数を最小化する。

9.5 更新順序

for each auxiliary update:
    fix prediction F
    optimize era scale sigma
    if df estimation is enabled:
        optimize nu
    evaluate exact objective
    if objective increased:
        restore previous sigma and nu

10.1 MAEとQuantileの特別処理

非微分可能objectiveでは、木構造探索用のpseudo-curvatureと 葉値のexact minimizerを分離する。 

1. pseudo-gradient and positive surrogate curvature are computed
2. tree structure is selected
3. for every fixed leaf:
       solve the exact one-dimensional leaf objective
4. apply coverage-dependent regularization
5. perform global exact-objective line search

非分離正則化が有効な場合、各葉の独立なmedianまたはquantileは 完全問題の厳密解ではないため、探索方向の初期値として使用する。

11. Binary Student-t Classification

\(z_i=F_i/s\)、\(p_i=T_\nu(z_i)\)、 \(d_i=f_\nu(z_i)/s\) とする。

\[ \ell_i = -y_i\log p_i-(1-y_i)\log(1-p_i), \] \[ g_i = \frac{(p_i-y_i)d_i} {p_i(1-p_i)}, \] \[ h_i^{\mathrm F} = \frac{d_i^2} {p_i(1-p_i)}. \]

11.1 数値安定化

  • \(y=1\)ではStudent-t log-CDFを直接計算
  • \(y=0\)ではlog-survival functionを直接計算
  • 確率は勾配計算時に数値範囲へclip
  • Fisher曲率に下限を加える
  • raw scoreの絶対値に任意の上限を設定可能

11.2 ラベル汚染

設定時は、

\[ \widetilde p_i = (1-\epsilon_+-\epsilon_-)p_i+\epsilon_- \]

を損失へ使用する。\(\epsilon_+,\epsilon_-\) は固定値または 制約付きvalidation推定値とし、木と同時に無制約最適化しない。

12. Ranking Objective

12.1 Pair構造

struct RankingPair {
    RowIndex preferred_row;
    RowIndex other_row;
    double label;
    double base_weight;
    double sampling_probability;
    EraId era;
    QueryId query;
};

pairは同一queryかつ同一era内に限定する。 tie targetは設定により除外するか、label \(1/2\) として扱う。

12.2 Student-t pairwise link

\(d_{ij}=F_i-F_j\)、\(s_e=\sqrt{2}\sigma_e\) とする。

\[ p_{ij}=T_\nu(d_{ij}/s_e). \]
\[ g_{ij}^{(d)} = \frac{ (p_{ij}-b_{ij})p_{ij}' }{ p_{ij}(1-p_{ij}) }, \qquad h_{ij}^{(d)} = \frac{(p_{ij}')^2} {p_{ij}(1-p_{ij})}. \]

サンプルへの集約は、

\[ g_i\mathrel{+}=w_{ij}g_{ij}^{(d)}, \qquad g_j\mathrel{-}=w_{ij}g_{ij}^{(d)}, \] \[ h_i\mathrel{+}=w_{ij}h_{ij}^{(d)}, \qquad h_j\mathrel{+}=w_{ij}h_{ij}^{(d)}. \]

12.3 Pair sampling

pair \(i,j\) の抽出確率を \(\pi_{ij}^{\mathrm{pair}}\) とし、 実効重みを、

\[ \widetilde w_{ij} = \frac{w_{ij}} {\max(\pi_{ij}^{\mathrm{pair}},\pi_{\min})} \]

とする。過大な逆確率重みを避けるため、次を提供する。

  • 抽出確率下限
  • 逆確率重み上限
  • 自己正規化importance weight
  • 一様pair samplingによる検証モード

12.4 LambdaRankモード

各boosting反復開始時に現在順位から \(|\Delta\mathrm{NDCG}_{ij}|\) を計算し、当該木の構築中は固定する。

LambdaRankモードは反復間でpair weightが変化する。 したがって全学習期間を通じた単一固定スカラーobjectiveの単調減少は主張しない。 line searchは当該反復で固定されたLambda surrogateに対して実行する。

13. Capped KL-DRO Solver

13.1 Primal問題

\[ \max_q \sum_eq_eL_e \]

制約は、

\[ q_e\geq0, \qquad \sum_eq_e=1, \qquad \sum_eq_e\log\frac{q_e}{\pi_e}\leq\rho, \qquad q_e\leq c_e=\kappa_e\pi_e. \]

13.2 実行可能性

最低条件は、

\[ \pi_e>0, \qquad c_e>0, \qquad \sum_ec_e\geq1, \qquad \rho\geq0. \]

さらに、cap付き単体上で達成可能な最小KL値が \(\rho\) 以下でなければならない。 通常、\(\pi_e\leq c_e\) が全eraで成立すれば \(q=\pi\) が実行可能である。

13.3 KKT形式

KL制約がactiveで双対温度 \(\tau>0\) の場合、 capへ到達していないeraの解は指数傾斜形になる。

\[ q_e^\star = \min \left[ c_e,\, \pi_e \exp\left( \frac{L_e-\eta}{\tau}-1 \right) \right]. \]

\(\eta\) は総和制約を満たす正規化双対変数である。 実装では定数 \(-1\) を \(\eta\) へ吸収し、

\[ q_e^\star = \min \left[ c_e,\, \pi_e\exp((L_e-\eta)/\tau) \right] \]

として扱う。

13.4 Solver手順

Input:
    era loss L
    reference weight pi
    KL radius rho
    cap c

Step 1:
    validate feasibility

Step 2:
    compute cap-constrained linear maximizer q_linear
    by allocating probability to eras in descending loss order

Step 3:
    if KL(q_linear || pi) <= rho:
        return q_linear
        KL constraint is inactive

Step 4:
    otherwise KL constraint is active
    find temperature tau > 0 by outer bisection

Step 5:
    for each candidate tau:
        find eta by inner bisection such that
        sum_e min(c_e, pi_e * exp((L_e - eta) / tau)) = 1

Step 6:
    compute q(tau), KL(q(tau) || pi)

Step 7:
    update tau until KL equals rho within tolerance

Step 8:
    return q, robust risk, KKT diagnostics

13.5 Overflow防止

指数はlog-domainで評価し、cap判定を先に行う。

\[ \log q_e^{\mathrm{uncapped}} = \log\pi_e+\frac{L_e-\eta}{\tau}. \]

\(\log q_e^{\mathrm{uncapped}}\geq\log c_e\) なら指数計算をせず \(q_e=c_e\) とする。

13.6 収束条件

  • 総和残差：\(|\sum_eq_e-1|\leq\varepsilon_{\mathrm{sum}}\)
  • KL残差：\(|D_{\mathrm{KL}}(q\Vert\pi)-\rho|\leq\varepsilon_{\mathrm{KL}}\)
  • cap違反：\(\max_e(q_e-c_e)\leq\varepsilon_{\mathrm{cap}}\)
  • 負重み：\(\min_eq_e\geq-\varepsilon_q\)

14. Era Downside Regularizer

固定benchmark lossを \(b_e\) とし、

\[ u_e=\frac{L_e-b_e}{s_d}, \qquad d_e=s_d\log(1+\exp u_e). \]
\[ R_{\mathrm{down}} = \sum_e\pi_ed_e^2. \]

\(\sigma(u)=1/(1+\exp(-u))\) とすると、

\[ \frac{\partial d_e}{\partial L_e} = \sigma(u_e), \] \[ \frac{\partial R_{\mathrm{down}}}{\partial L_e} = 2\pi_ed_e\sigma(u_e). \]

サンプル予測への勾配は、

\[ \frac{\partial R_{\mathrm{down}}}{\partial F_i} = 2\pi_{e_i}d_{e_i}\sigma(u_{e_i}) a_i g_i^{\mathrm{task}}. \]

木構築用曲率にはGauss-Newton近似を用い、 負になる可能性のある二階chain termを含めない。 完全objectiveの正当性はline searchで検証する。

15. Factor Penalty

15.1 Era演算子構築

era \(e\) の重みを正規化し、\(W_e=\operatorname{diag}(a_i)\) とする。

\[ C_e = I-\mathbf1 \frac{\mathbf1^\top W_e} {\mathbf1^\top W_e\mathbf1}. \]

中心化因子は、

\[ \widetilde Z_e=C_eZ_e. \]

ridge付き因子射影は、

\[ M_e = \widetilde Z_e^\top W_e\widetilde Z_e+\gamma_ZI, \] \[ P_e = \widetilde Z_eM_e^{-1}\widetilde Z_e^\top W_e. \]

15.2 ペナルティ

\[ R_{\mathrm{factor}} = \frac12 \sum_e \beta_e (C_eF_e)^\top W_eP_e(C_eF_e), \qquad \beta_e=\frac{\pi_e}{s_{F,e}^2}. \]

15.3 行列を明示構築しない計算

for each era e:
    centered_prediction = weighted_center(F_e)
    factor_rhs = Z_tilde_e^T * W_e * centered_prediction
    factor_coefficient = solve(M_e, factor_rhs)
    projected_prediction = Z_tilde_e * factor_coefficient
    penalty += 0.5 * beta_e
               * centered_prediction^T
               * W_e
               * projected_prediction

    gradient_e = beta_e
                 * C_e^T
                 * W_e
                 * projected_prediction

15.4 曲率majorizer

完全Hessianを明示保存せず、行和上界またはLanczos推定最大固有値を用いる。

\[ H_{\mathrm{factor},e} = \beta_eC_e^\top W_eP_eC_e \succeq0. \]

対角majorizerは、

\[ d_i^{\mathrm{factor}} = H_{ii} + \sum_{j\neq i}|H_{ij}|. \]

大規模実装では安全な上界 \(\lambda_{\max}(H_{\mathrm{factor},e})\) を era内全行へ割り当てるscalar majorizerも提供する。

15.5 Rank deficiency

  • 定数列を除去する
  • ほぼゼロ分散のfactorを除去する
  • pivoted QRまたはSVDでrankを診断する
  • \(\gamma_ZI\) により逆問題を安定化する
  • 条件数が上限を超える場合は警告する

16. Turnover Penalty

16.1 Portfolio operator

当該boosting反復中に固定された疎行列 \(B_e\) から、

\[ p_e=B_eF_e \]

を計算する。

B_eは以下を含められる。

  • era内加重中心化
  • ex-ante volatility scaling
  • 固定factor projection
  • 流動性に基づく固定係数

16.2 時系列差分operator

全eraを結合した予測を \(F\) とし、

\[ d=DF \]

と表す。\(D\) はportfolio変換とinstrument時系列差分を合成した疎行列である。

16.3 Smooth-L1

\[ R_{\mathrm{turn}}(F) = \sum_kc_k \left( \sqrt{d_k^2+\delta^2}-\delta \right). \]

差分空間の勾配は、

\[ u_k = c_k\frac{d_k}{\sqrt{d_k^2+\delta^2}}. \]

予測空間の勾配は、

\[ \nabla_FR_{\mathrm{turn}} = D^\top u. \]

差分空間の曲率は、

\[ w_k = c_k \frac{\delta^2} {(d_k^2+\delta^2)^{3/2}}. \]

Hessianは、

\[ H_{\mathrm{turn}} = D^\top\operatorname{diag}(w)D \succeq0. \]

16.4 Instrument欠落

  • 前eraに存在しないinstrument：新規entryとして差分対象外またはzero baselineと比較
  • 現eraに存在しないinstrument：exit costを扱う場合のみzero positionとの差を追加
  • 複数期間欠落：直接の直前観測と比較するか、連続eraだけに限定

方式はturnover_missing_policyとしてモデルへ保存する。

17. GradientCurvatureComposer

17.1 合成勾配

\[ \begin{aligned} G_i={}& q_{e_i}^\star a_i g_i^{\mathrm{task}} + \lambda_{\mathrm{down}}g_i^{\mathrm{down}} \\ &+ \lambda_{\mathrm{factor}}g_i^{\mathrm{factor}} + \lambda_{\mathrm{turn}}g_i^{\mathrm{turn}}. \end{aligned} \]

17.2 合成曲率

\[ H_i = q_{e_i}^\star a_i h_i^{\mathrm{task}} + \lambda_{\mathrm{down}}\widetilde h_i^{\mathrm{down}} + \lambda_{\mathrm{factor}}\widetilde h_i^{\mathrm{factor}} + \lambda_{\mathrm{turn}}\widetilde h_i^{\mathrm{turn}} + h_{\min}. \]

\(\widetilde h\)は非負のGauss-Newton曲率または対角majorizerである。

17.3 スケーリング

taskごとの勾配scale差による不安定性を避けるため、 任意のgradient normalizationを提供するが、標準では無効とする。 有効化する場合はera内共通倍率だけを使用し、サンプル間相対重みを変えない。

17.4 出力検証

  • 全勾配が有限値
  • 全曲率が正
  • 総曲率が下限以上
  • 勾配絶対値上位サンプルとeraを診断ログへ保存

19. Coverage-Dependent Leaf Prior

19.1 曲率質量分布

\[ H_A=\sum_eH_{A,e}, \qquad u_{A,e} = \frac{H_{A,e}}{H_A}. \]

\(H_A>0\)を葉候補条件として要求する。

19.2 Effective era数

\[ N_{\mathrm{eff},A} = \frac{1}{\sum_eu_{A,e}^2}. \]

19.3 基準era分布との差

\[ \chi_A^2 = \sum_{e\in\mathcal E_A} \frac{(u_{A,e}-\pi_{A,e})^2} {\pi_{A,e}+\varepsilon}. \]

19.4 鮮度不足

\[ d_A^{\mathrm{stale}} = 1- \sum_eu_{A,e} \exp[-\lambda_d(t_{\max}-t_e)]. \]

19.5 Prior precision

\[ \lambda_{A,\mathrm{cov}} = H_{\mathrm{ref}} \left[ \frac{\kappa_E} {N_{\mathrm{eff},A}+\varepsilon} + \kappa_\chi\chi_A^2 + \kappa_dd_A^{\mathrm{stale}} \right]. \]

\(H_{\mathrm{ref}}\)はrootまたは全データの平均曲率scaleから固定する。 候補葉ごとに異なる値を計算するが、木確定後は当該木の構造正則化係数として凍結する。

19.6 葉値

\[ v_A^\star = - \frac{ \operatorname{SoftThreshold}(G_A,\lambda_1) }{ H_A+\lambda_2+\lambda_{A,\mathrm{cov}} }.

19.7 Priorの確率解釈

条件付きpriorを、

\[ v_A \mid \mathcal S_A \sim \mathcal N \left( 0, \lambda_{A,\mathrm{cov}}^{-1} \right) \]

と解釈する。\(\mathcal S_A\) は葉のera support統計である。 低coverage、基準分布からの集中、古いeraへの偏りほどprior分散が小さくなる。

20. Robust Split Evaluator

20.1 Era別surrogate改善

\[ \begin{aligned} \Delta_e={}& G_{P,e}v_P+\frac12H_{P,e}v_P^2 \\ &- G_{L,e}v_L-\frac12H_{L,e}v_L^2 \\ &- G_{R,e}v_R-\frac12H_{R,e}v_R^2. \end{aligned} \]

20.2 Split用ambiguity set

\[ \mathcal Q_{\mathrm{split}} = \left\{ q: q\geq0,\; \sum_eq_e=1,\; D_{\mathrm{KL}}(q\Vert\pi_P)\leq\rho_s,\; q_e\leq\kappa_s\pi_{P,e} \right\}. \]

20.3 下側robust gain

\[ G_{\mathrm{rob}} = \min_{q\in\mathcal Q_{\mathrm{split}}} \sum_eq_e\Delta_e. \]

これは損失DROの最大化問題へ \(L_e=-\Delta_e\) を渡し、

\[ G_{\mathrm{rob}} = - \max_{q\in\mathcal Q_{\mathrm{split}}} \sum_eq_e(-\Delta_e) \]

として同じsolverを再利用する。

20.4 Dispersion penalty

加重medianを \(m_\Delta\) とし、

\[ \operatorname{MAD}_{\pi_P}(\Delta) = \operatorname{WeightedMedian}_{\pi_P} \left( |\Delta_e-m_\Delta| \right). \]

20.5 Negative-era penalty

\[ R_{\mathrm{neg}} = \sum_e\pi_{P,e}\max(0,-\Delta_e). \]

20.6 構造正則化差

\[ \Delta\Omega = \Omega_L(v_L)+\Omega_R(v_R)-\Omega_P(v_P). \]

葉数penaltyを別途差し引く場合、\(\Delta\Omega\)へ二重計上しない。

20.7 最終score

\[ \operatorname{Score} = G_{\mathrm{rob}} - \Delta\Omega - \gamma_{\mathrm{split}} - \kappa_{\mathrm{neg}}R_{\mathrm{neg}} - \kappa_{\mathrm{disp}} \operatorname{MAD}_{\pi_P}(\Delta). \]

20.8 Candidate評価手順

for each split candidate:
    compute aggregate left and right statistics
    reject if curvature or count constraints fail

    compute era-specific left and right statistics
    compute effective era coverage
    reject if coverage constraints fail

    compute coverage prior precision for parent, left, right
    compute parent, left, right leaf values

    compute era improvement vector delta
    solve lower-tail capped KL-DRO
    compute negative-era penalty
    compute weighted MAD
    compute structural regularization difference

    score candidate
    retain candidate if score is the current maximum

22.3 時間減衰

状態を時刻 \(t_0\) から \(t_1\) へ移すとき、

\[ \operatorname{decay}(t_0,t_1) = \exp[-\lambda_t(t_1-t_0)]. \]

  • online_robust：有界scoreによるオンラインM推定

22.4 Prior smoothing

\[ \operatorname{OTS}_{i,c} = \frac{ a_c\mu_0+n_{i,c}\widehat\mu_{i,c} }{ a_c+n_{i,c} }. \]

countが小さいカテゴリはglobal priorへ縮小する。 \(a_c\)はカテゴリ階層、cardinalityまたはvalidationで決定する。

22.5 学習時cross-fitting

OTS自体はstrict-pastで生成する。 ordered boosting residualを使う場合は、さらにpurged expanding-window foldで out-of-time予測を生成する。 

for each temporal fold k:
    fit prefix model on train window T_k
    predict validation window V_k
    store out-of-time prediction

construct ordered pseudo-residuals from out-of-time predictions
train final ensemble using those residual-safe features or statistics

23. Exact Objective Evaluator

23.1 評価対象

\[ \begin{aligned} J(F)={}& R_{\mathrm{DRO}}(F) + \lambda_{\mathrm{down}}R_{\mathrm{down}}(F) \\ &+ \lambda_{\mathrm{factor}}R_{\mathrm{factor}}(F) + \lambda_{\mathrm{turn}}R_{\mathrm{turn}}(F) \\ &+ R_{\mathrm{scale}}(\sigma) + \Omega_{\mathrm{ensemble}}. \end{aligned} \]

23.2 構造正則化の凍結

候補木について計算された次の値は、木のline search中に固定する。

  • 木構造
  • 各葉のcoverage prior precision
  • 各葉のbase value
  • 葉数および深さpenalty

step \(\alpha\) に対する葉値は \(\alpha v_A\) となり、 L1・L2正則化は実際のscaled leaf valueで評価する。

23.3 Exactの意味

次を近似せずに再計算する。

  • 各サンプルtask loss
  • 各era loss
  • DRO内側最適化
  • downside penalty
  • factor penalty
  • turnover penalty
  • 候補木の実際のL1・L2正則化

数値特殊関数およびsolver toleranceによる有限精度誤差は存在する。

24. Line Search

24.1 探索問題

\[ \alpha^\star = \arg\min_{0\leq\alpha\leq\alpha_{\max}} J(F+\alpha f). \]

24.2 Armijo方向微分

\[ d_0 = \nabla J(F)^\top f. \]

\(d_0\geq0\) の場合、候補木は下降方向ではないため原則棄却する。 数値誤差許容範囲内なら小stepの直接評価を一度だけ許可できる。

24.3 Backtracking

alpha = min(base_learning_rate, alpha_max)
current_objective = evaluate(F)

while alpha >= minimum_alpha:
    candidate_prediction = F + alpha * tree_prediction

    candidate_objective = evaluate_exact_objective(
        candidate_prediction,
        reoptimize_dro=True,
        fixed_objective_state=True,
        fixed_tree_regularization=True,
    )

    if candidate_objective
       <= current_objective + armijo_c1 * alpha * directional_derivative:
        accept tree with step alpha
        return accepted

    alpha = alpha * backtracking_factor

reject tree
return rejected

24.4 補助パラメータとの関係

木のline search中は \(\sigma,\nu\) を固定する。 木受理後、補助パラメータをblock updateし、 その更新でも完全objectiveが増加しないことを確認する。

36. 数学テスト設計

36.1 Student-t

  • 解析勾配と中心有限差分の一致
  • 自由度勾配と有限差分の一致
  • MM surrogateが元損失以上であること
  • 現在点でMM surrogateと元損失が接すること
  • 影響関数が有界で尾部で0へ近づくこと

36.2 DRO

  • 小規模問題で汎用凸solverと一致
  • capなしの場合に指数傾斜解と一致
  • 半径0で \(q=\pi\)
  • 十分大きい半径でcap付き線形最大化解と一致
  • KKT相補性条件を満たす
  • lossを増加したeraの最悪重みが原則として減少しない

36.3 Factor penalty

  • ペナルティが非負
  • factor直交予測でほぼ0
  • 解析勾配と有限差分が一致
  • Hessianの最小固有値が許容誤差以上

36.4 Turnover

  • 定数position系列で0
  • 解析勾配と有限差分が一致
  • Hessianが半正定値
  • \(\delta\to0\)でL1へ近づく

36.5 Leaf optimization

  • soft-threshold解が部分勾配最適条件を満たす
  • coverage低下に対して葉値絶対値が非増加
  • L1増加に対して葉値絶対値が非増加

37. Leakageテスト設計

  1. 未来targetを書き換えても過去OTSが変化しないこと
  2. 同一era targetを書き換えても当該era OTSが変化しないこと
  3. 未成熟targetがstateへ追加されないこと
  4. embargo境界内targetが除外されること
  5. validation targetがbinningへ影響しないこと
  6. validation targetがfactor operatorへ影響しないこと
  7. 過去時点stateから同じ過去予測を再現できること

38. Tree・統合テスト設計

38.1 Robust split synthetic test

次の人工特徴を作る。

  • 特徴A：全eraで小さく一貫したsignal
  • 特徴B：単一eraで非常に大きいsignal
  • 特徴C：既知factorそのもの
  • 特徴D：頻繁な時間反転を生むsignal

期待結果は次である。

  • 標準gainはBを選び得る
  • robust splitは設定範囲でAを優先する
  • factor penalty有効時にCの選択率が低下する
  • turnover penalty有効時にDの葉値が縮小する

38.2 Exact objective test

全受理木について、

\[ J(F^{(m)}) \leq J(F^{(m-1)}) + \varepsilon_J \]

を検証する。LambdaRankのような反復変動surrogateでは、 同一反復内で固定したobjectiveに対して検証する。

38.3 Serialization test

\[ \max_i \left| \widehat F_i^{\mathrm{before}} - \widehat F_i^{\mathrm{after}} \right| \leq \varepsilon_{\mathrm{serialization}}. \]

42. 詳細受入基準

  1. Student-tの解析勾配が有限差分と指定誤差内で一致すること。
  2. Student-t MM surrogateが試験範囲内で常に元損失を上回り、 現在点で一致すること。
  3. capped KL-DROが小規模参照solverと一致し、KKT残差条件を満たすこと。
  4. strict-past ordered encodingが未来target変更に対して不変であること。
  5. robust splitが低era coverage候補を設定どおり拒否すること。
  6. coverage priorによる葉値が定義した凸葉部分問題の解であること。
  7. factor penaltyおよびturnover penaltyの勾配が有限差分と一致すること。
  8. 全受理木について対象となる完全objectiveが許容誤差内で非増加であること。
  9. 回帰、二値分類、pairwise ranking、quantileのfit・predictが完了すること。
  10. purged walk-forwardが未成熟targetとembargo対象をtrainから除外すること。
  11. 保存前後の予測、objective state、ordered stateが一致すること。
  12. 決定論モードで同一入力・同一設定・同一thread数のモデルhashが一致すること。

43. 最終実装仕様

FORT-GBM v2は、木構造探索を単純な全サンプル平均利得へ依存させない。 各反復でStudent-t MMまたはtask固有の正曲率を構成し、 capped era-DROによる敵対era重み、factor・turnover・downsideの勾配を合成する。 分割候補はera別改善量の下側robust gainで評価し、 低era coverageの葉には明示的なGaussian prior precisionを加える。 生成された木は探索方向としてのみ扱い、DROを再最適化した完全目的関数上の line searchを通過した場合に限ってensembleへ追加する。
\[ \boxed{ \begin{aligned} \mathrm{FORT\mbox{-}GBM\ v2} ={}& \mathrm{Student\mbox{-}t\ MM} + \mathrm{Point\mbox{-}in\mbox{-}Time\ Ordered\ Encoding} \\ &+ \mathrm{Capped\ Era\ KL\mbox{-}DRO} + \mathrm{Robust\ Split} \\ &+ \mathrm{Coverage\ Leaf\ Prior} + \mathrm{Factor\ Penalty} \\ &+ \mathrm{Turnover\ Penalty} + \mathrm{Exact\ Acceptance\ Test}. \end{aligned} } \]
